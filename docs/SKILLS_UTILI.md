# Skill consigliate e installate

Questa selezione è stata fatta partendo dal catalogo **curated** ufficiale delle skill Codex (repository `openai/skills`) e scegliendo quelle più utili in modo trasversale per sviluppo, debugging, documentazione e produttività.

## Criteri di scelta

- Utilità generale in molti progetti (non legate a un singolo framework).
- Supporto a task frequenti: analisi documenti, test UI, fix CI, sicurezza.
- Buon rapporto tra facilità d'uso e impatto pratico.

## Skill installate

1. **doc**
   - A cosa serve: estrazione/lettura strutturata di documentazione tecnica.
   - Quando usarla: onboarding su repo nuovi, sintesi di specifiche, analisi README lunghi.

2. **pdf**
   - A cosa serve: lavorare con contenuti in PDF.
   - Quando usarla: analisi requisiti forniti in PDF, confronto tra versioni di documenti.

3. **playwright**
   - A cosa serve: automazione browser per test end-to-end e verifiche funzionali.
   - Quando usarla: validare flussi utente web, regressioni UI, smoke test rapidi.

4. **screenshot**
   - A cosa serve: catturare screenshot in modo ripetibile.
   - Quando usarla: documentare cambi UI, allegare evidenze visive in issue/PR.

5. **gh-fix-ci**
   - A cosa serve: supporto nella diagnosi e risoluzione di pipeline CI fallite.
   - Quando usarla: build/test rotti su GitHub Actions, errori intermittenti di workflow.

6. **security-best-practices**
   - A cosa serve: checklist e linee guida pratiche di sicurezza applicativa.
   - Quando usarla: hardening pre-release, review di configurazioni e dipendenze.

7. **openai-docs**
   - A cosa serve: consultazione guidata della documentazione OpenAI.
   - Quando usarla: integrazioni API, aggiornamento pattern/SDK, troubleshooting.

8. **spreadsheet**
   - A cosa serve: lettura e trasformazione di dati tabellari.
   - Quando usarla: report veloci, analisi CSV/XLSX, pulizia dati operativa.

9. **transcribe**
   - A cosa serve: trascrizione contenuti audio.
   - Quando usarla: meeting notes, interviste utenti, conversione audio->testo.

10. **imagegen**
    - A cosa serve: generazione immagini per prototipi o asset di supporto.
    - Quando usarla: mockup rapidi, supporto visuale a documentazione o presentazioni.

## Nota operativa

Dopo l'installazione di nuove skill, potrebbe essere necessario **riavviare Codex** per renderle disponibili nella sessione corrente.
